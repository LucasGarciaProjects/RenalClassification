KIDNEY ULTRASOUND ANALYSIS PROJECT
==========================================

OVERVIEW
--------

This project consists of two main components: the model training notebooks and the web application for kidney ultrasound image analysis. It uses deep learning models to classify ultrasound kidney images as either healthy or diseased. Below is a description of the core files and folders included in the project:

Project Structure and Key Files

EfficientNetB0.ipynb, MobileNetV2.ipynb, ResNet50.ipynb:
Jupyter notebooks containing the full training, fine-tuning, evaluation, and threshold optimization process for each of the three architectures. These notebooks document the progression from baseline models to improved versions capable of distinguishing between healthy and diseased kidneys.

FINAL_CONCLUSIONS.ipynb:
A final notebook that summarizes the performance of the three trained models, includes visual comparisons (ROC curves, bar charts, F1-score tables), and outlines conclusions and future directions for the project.

final_models/:
This folder contains the best-performing trained model (.keras) for each architecture, saved after threshold optimization and fine-tuning. These are the models integrated into the web application for real-time predictions.

requirements.txt:
Lists all required Python packages and their versions for running the backend of the web application, including TensorFlow, OpenCV, and ReportLab.

images_with_mask/ (generated by preprocessing script):
Contains kidney ultrasound images with their corresponding region-of-interest masks applied. This directory is created by running the following script, which overlays the binary mask from .mat files onto the original grayscale image:

"""
import os
import cv2
import numpy as np
from scipy.io import loadmat

raw_images_dir = "data/raw/images"
raw_masks_dir = "data/raw/masks"
output_dir = "data/processed/images_with_mask"

os.makedirs(output_dir, exist_ok=True)
image_files = [f for f in os.listdir(raw_images_dir) if f.lower().endswith(('.jpg', '.png', '.jpeg'))]

for img_file in image_files:
    base_name = os.path.splitext(img_file)[0]
    mask_file = f"{base_name}.mat"
    mask_path = os.path.join(raw_masks_dir, mask_file)
    
    if os.path.exists(mask_path):
        img_path = os.path.join(raw_images_dir, img_file)
        img = cv2.imread(img_path, cv2.IMREAD_GRAYSCALE)
        mask = loadmat(mask_path)['mask']
        
        if mask.shape != img.shape:
            mask = cv2.resize(mask, (img.shape[1], img.shape[0]))
        
        img_masked = img * mask
        img_masked_rgb = cv2.cvtColor(img_masked, cv2.COLOR_GRAY2RGB)
        output_path = os.path.join(output_dir, img_file)
        cv2.imwrite(output_path, img_masked_rgb)

print("All images saved in", output_dir)

KIDNEY ULTRASOUND ANALYSIS WEB APPLICATION
==========================================

OVERVIEW
--------
This is a web application for kidney ultrasound image analysis using deep learning models. 
The system provides automated classification of kidney ultrasound images as either "healthy" 
or "pathological" using three different pre-trained neural network models.

APPLICATION FLOW
---------------
1. User Interface (Frontend):
   - Users upload kidney ultrasound images and corresponding mask files
   - Select one of three available deep learning models
   - View real-time analysis results with confidence scores
   - Generate and download PDF reports

2. Backend Processing:
   - Image preprocessing with mask application for ROI extraction
   - Model inference using TensorFlow/Keras
   - Label correction for specific models (EfficientNet/ResNet)
   - Base64 encoding for image transmission
   - PDF report generation with analysis results

3. Model Integration:
   - MobileNetV2 (Fine-tuned) - maintains original labels
   - EfficientNetB0 (Fine-tuned) - labels are inverted
   - ResNet (Fine-tuned) - labels are inverted
   - All models use the same preprocessing pipeline

WEB APPLICATION KEY COMPONENTS
--------------

Frontend (Next.js/React):
- FileUpload component: Handles image and mask file uploads
- Results component: Displays prediction results and images
- Toolbar component: Model selection and theme controls
- API routes: Communication with Python backend

Backend (Python):
- predict.py: Main inference script with model loading and prediction
- generate_pdf.py: PDF report generation with analysis results
- Model caching system for performance optimization
- Image preprocessing with mask application

API Routes:
- /api/predict: Handles file uploads and returns prediction results
- /api/generate-pdf: Creates downloadable PDF reports

IMPORTANT FILES
---------------

Frontend:
- app/page.tsx: Main application page
- app/components/: React components for UI
- app/api/: Next.js API routes for backend communication

Backend:
- server/scripts/predict.py: Core prediction logic and model management
- server/scripts/generate_pdf.py: PDF report generation
- models/: Directory containing .keras model files

Configuration:
- package.json: Frontend dependencies and scripts
- requirements.txt: Python dependencies
- next.config.js: Next.js configuration

SETUP AND EXECUTION
-------------------
1. Install frontend dependencies: npm install
2. Install Python dependencies: pip install -r requirements.txt
3. Place model files in the models/ directory
4. Start frontend: npm run dev (port 3000)
5. Access application at http://localhost:3000

TECHNOLOGIES USED
-----------------
- Frontend: Next.js, React, TypeScript
- Backend: Python, TensorFlow/Keras, OpenCV, ReportLab
- Communication: REST API, Base64 encoding
- File handling: Temporary file management with cleanup

This application demonstrates a complete pipeline for medical image analysis 
from user interface to automated report generation using modern web technologies 
and deep learning.
